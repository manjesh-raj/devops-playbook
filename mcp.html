Module 01

What is MCP?

The open protocol that makes AI stop reinventing integrations — released by Anthropic, Nov 2024. Now adopted by OpenAI and Google DeepMind.

🧠 ELI5 — The USB-C Analogy

MCP is the USB-C of AI tools. Before USB-C, every device had a different charger — nightmare. USB-C standardized it all. Before MCP, every AI tool needed custom code to talk to every data source. MCP standardizes how AI connects to everything. Write one MCP server for your Bitbucket instance and any AI that speaks MCP can use it instantly.

⚡ The N×M Problem — Before and After MCP
❌ Before MCP: N×M integrations

5 data sources × 3 AI tools = 15 custom integrations. Each one had different auth, different error handling, duplicate code. Change one API and 3 integrations break.

Claude → GitHub (custom)
Claude → Postgres (custom)
Claude → Jira (custom)
GPT-4 → GitHub (custom)
GPT-4 → Postgres (custom)
GPT-4 → Jira (custom)
... 9 more ...
✅ After MCP: N+M integrations

5 data sources × 1 MCP standard = 5 MCP servers. Any AI that speaks MCP connects to all 5 instantly. Build once, use everywhere.

Any AI → MCP Client → GitHub MCP
Any AI → MCP Client → Postgres MCP
Any AI → MCP Client → Jira MCP
Add a new AI? Zero new integrations.
Add a new tool? One MCP server.
📅 Timeline — How We Got Here
2023
Function calling — LLMs can call tools, but schemas are baked into every single API request. No standardization. Every provider did it differently.
Nov 2024
Anthropic releases MCP as an open standard. Tools are described once per session (not per request). Claude Desktop ships with MCP support out of the box.
Early 2025
OpenAI and Google DeepMind adopt MCP. Cursor, Windsurf, VS Code Copilot, and LangGraph all add native MCP support. 100+ official MCP servers published.
2025+
MCP becomes the de-facto standard for AI tool integration. Every major enterprise SaaS (Atlassian, AWS, Slack) ships an official MCP server.
🐳 DevOps Analogies — You Already Know This
🐳 MCP is like Docker
Docker standardized app packaging — "build once, run anywhere". MCP standardizes AI tool integration — "build once, use with any AI".
📡 MCP is like HTTP
HTTP is a protocol, not a product. Anyone can implement it. MCP is the same — an open spec. Any AI provider, any tool vendor can implement it.
🔌 MCP is like a K8s Operator
An Operator defines how a system behaves and reconciles state. An MCP server defines what an AI can do and executes actions on its behalf.
📦 MCP is like npm/PyPI
Instead of writing code to use a library, you install an MCP server. Your AI gets all the tools instantly — no integration code required.

Core Concepts

Three participants, three primitives — that's all you need to understand MCP completely.

🍽️ ELI5 — The Restaurant Model

Think of a restaurant. The Host is the restaurant manager (Claude Desktop or your app) — responsible for everything. The Client is a waiter who knows exactly one kitchen. The Server is a kitchen that specializes in one type of food (file system, GitHub, Postgres). The manager can have multiple waiters, each connecting to a different kitchen simultaneously.

Host — the application the user runs. Creates and manages Clients. Has access to the LLM. Examples: Claude Desktop, your custom Python agent, Cursor.
Client — lives inside the Host. Maintains exactly one connection to one Server. Handles the JSON-RPC protocol. Usually invisible to you.
Server — a lightweight process that exposes tools and data. Can be local (stdin/stdout) or remote (HTTPS). You build these in Python with FastMCP.
🧩 The Three Primitives
ACTION
🔧 Tool
What: Callable functions — the AI invokes these to do things.

DevOps analogy: Like a kubectl command or REST API endpoint.

Examples: list_pods(), create_pr(), query_db(), restart_service()
READ-ONLY
📄 Resource
What: Read-only data — the AI reads this for context.

DevOps analogy: Like a Prometheus metric, a log file, or a config file.

Examples: system://info, db://schema, current cluster state
TEMPLATE
📝 Prompt
What: Reusable interaction templates for structured tasks.

DevOps analogy: Like a runbook or operational playbook.

Examples: "Diagnose this service", "Review this PR", "Summarize incidents"
🔬 Advanced Features — Sampling & Elicitation
Sampling
A Server can ask the LLM "what should I do next?" — allows servers to request AI reasoning mid-execution. Example: your Bitbucket MCP server gets a complex PR diff and asks Claude to summarize it before posting a comment.
Elicitation
A Server can ask the user "is it OK to do X?" — enables human-in-the-loop approvals. Example: "This will restart the production pod. Are you sure?" before executing a destructive action.

Module 03

How MCP Works Under the Hood

JSON-RPC 2.0, two transports, five lifecycle steps. This is the entire protocol.

🧠 ELI5 — Very Formal Conversation

MCP uses JSON-RPC 2.0 — like HTTP but for function calls. Every message is JSON, every request gets a response. It's a very formal conversation: "Hello" → "I support X and Y" → "List tools" → "Here are my 6 tools" → "Call tool get_cpu_usage" → "CPU is 87%". That's it. Every MCP interaction follows this exact pattern.

📬 JSON-RPC Message Anatomy
Request (Client → Server)
{"jsonrpc": "2.0",
 "id": 3,
 "method": "tools/call",
 "params": {
   "name": "get_pipeline_status",
   "arguments": {
     "repository": "sre-autopilot",
     "limit": 5
   }
 }}
Response (Server → Client)
{"jsonrpc": "2.0",
 "id": 3,
 "result": {
   "content": [{
     "type": "text",
     "text": "[{\"build\":42,\"state\":\"SUCCESSFUL\",
       \"branch\":\"main\"}]"
   }]
 }}
Notification (Server → Client, no reply expected)
{"jsonrpc": "2.0",
 "method": "notifications/tools/list_changed"
 // No "id" field = notification, not a request
}
Error Response
{"jsonrpc": "2.0",
 "id": 3,
 "error": {
   "code": -32602,
   "message": "Invalid params",
   "data": {"missing": "repository"}
 }}

Module 05

Build Your First MCP Server

From zero to a working server in 15 minutes using FastMCP — the FastAPI of MCP.

⚡ Why FastMCP?

FastMCP is to MCP what FastAPI is to REST APIs. You write Python functions. It handles the entire JSON-RPC protocol, message routing, capability negotiation, and error handling. You just add @mcp.tool() and you're done. No boilerplate, no manual JSON construction.

📦 Step 1 — Install FastMCP
pip install fastmcp psutil
🔧 Step 2 — Write Your First Server
# system_monitor.py
from fastmcp import FastMCP
import psutil, socket, json

mcp = FastMCP("system-monitor")   # Give your server a name

@mcp.tool()
def get_cpu_usage() -> str:
    """Get current CPU usage percentage.
    Use this when the user asks about CPU load or performance."""
    return f"CPU: {psutil.cpu_percent(interval=1)}%"

@mcp.tool()
def get_memory_usage() -> str:
    """Get current memory usage statistics."""
    mem = psutil.virtual_memory()
    return f"Memory: {mem.percent}% used ({mem.used/1e9:.1f}GB / {mem.total/1e9:.1f}GB)"

@mcp.tool()
def get_disk_usage(path: str = "/") -> str:
    """Get disk usage for a specific path.
    Args:
        path: The filesystem path to check (default: /)
    """
    disk = psutil.disk_usage(path)
    return f"Disk {path}: {disk.percent}% used ({disk.free/1e9:.1f}GB free)"

@mcp.tool()
def get_top_processes(sort_by: str = "cpu", limit: int = 5) -> str:
    """Get top processes sorted by CPU or memory usage.
    Args:
        sort_by: Sort by 'cpu' or 'memory'
        limit: Number of processes to return (default 5)
    """
    procs = []
    for p in psutil.process_iter(['pid', 'name', 'cpu_percent', 'memory_percent']):
        try: procs.append(p.info)
        except: pass
    key = 'cpu_percent' if sort_by == 'cpu' else 'memory_percent'
    procs.sort(key=lambda x: x.get(key, 0), reverse=True)
    return json.dumps(procs[:limit])

if __name__ == "__main__":
    mcp.run()   # Uses STDIO transport by default
📊 Step 3 — Add a Resource (Read-only Context)
# Add this to system_monitor.py
@mcp.resource("system://info")
def get_system_info() -> str:
    """Read-only system info — AI can read this for context without calling a tool."""
    return json.dumps({
        "hostname": socket.gethostname(),
        "cpu_count": psutil.cpu_count(),
        "platform": psutil.sys.platform if hasattr(psutil, 'sys') else "linux",
        "total_memory_gb": round(psutil.virtual_memory().total / 1e9, 1)
    })

📐 MCP Server Design Principles
✅ Clear tool descriptions — the LLM reads your docstrings to decide which tool to call. Bad docstrings = wrong tool calls.
✅ Flat input schemas — avoid deeply nested objects. Simple types (str, int, bool) work best. LLMs struggle with complex nested schemas.
✅ Structured errors — return {"error": "message", "suggestion": "try X instead"} rather than letting exceptions propagate. The LLM can act on error messages.
✅ Idempotency — same input should produce same result. Safe to retry if something goes wrong during the agent loop.
✅ Rate limiting — add time.sleep(0.5) between external API calls. Your server is called by an LLM in a loop — it will hammer APIs without this.

# mcp_agent.py
import asyncio, os
from langchain_mcp_adapters.client import MultiServerMCPClient
from langchain_anthropic import ChatAnthropic
from langgraph.prebuilt import create_react_agent

async def main():
    # Step 1: Configure which MCP servers to connect to
    mcp_config = {
        "filesystem": {
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp", "/var/log"],
            "transport": "stdio"
        },
        "github": {
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-github"],
            "env": {"GITHUB_PERSONAL_ACCESS_TOKEN": os.getenv("GITHUB_TOKEN")},
            "transport": "stdio"
        }
    }

    # Step 2: Connect to servers and get all tools
    async with MultiServerMCPClient(mcp_config) as mcp:
        tools = await mcp.get_tools()
        print(f"✅ Loaded {len(tools)} tools from {len(mcp_config)} MCP servers")
        for t in tools: print(f"   • {t.name}")

        # Step 3: Create a LangGraph ReAct agent with MCP tools
        llm   = ChatAnthropic(model="claude-sonnet-4-6")
        agent = create_react_agent(llm, tools)

        # Step 4: Run it
        result = await agent.ainvoke({
            "messages": [{
                "role": "user",
                "content": "Check if there are any error logs in /var/log/app.log from the last hour"
            }]
        })

        print("\n📋 Agent Response:")
        print(result["messages"][-1].content)

asyncio.run(main())

🛠️ Complete Bitbucket MCP Server — 6 Tools
# bitbucket_mcp.py
from fastmcp import FastMCP
import requests, os, json
from base64 import b64encode

mcp = FastMCP("bitbucket-mcp")

# ── Configuration from environment variables ──────────────────────
BBHOST    = os.getenv("BITBUCKET_HOST", "https://api.bitbucket.org/2.0")
WORKSPACE = os.getenv("BITBUCKET_WORKSPACE")

def auth_headers():
    """Build auth headers — tries Personal Access Token first, falls back to Basic."""
    token = os.getenv("BITBUCKET_TOKEN")
    if token:
        return {"Authorization": f"Bearer {token}"}
    user = os.getenv("BITBUCKET_USERNAME")
    pwd  = os.getenv("BITBUCKET_PASSWORD")
    creds = b64encode(f"{user}:{pwd}".encode()).decode()
    return {"Authorization": f"Basic {creds}"}

# ── Tool 1: List Repositories ─────────────────────────────────────
@mcp.tool()
def list_repositories(workspace: str = None, pagelen: int = 10) -> str:
    """List repositories in a Bitbucket workspace.
    Args:
        workspace: Workspace slug (uses env var BITBUCKET_WORKSPACE if not provided)
        pagelen: Number of repos to return (default 10)
    """
    ws = workspace or WORKSPACE
    r = requests.get(f"{BBHOST}/repositories/{ws}",
                     headers=auth_headers(),
                     params={"pagelen": pagelen}, timeout=30)
    r.raise_for_status()
    repos = [{"name": x["name"], "slug": x["slug"], "language": x.get("language")}
             for x in r.json().get("values", [])]
    return json.dumps(repos)

# ── Tool 2: Get Pull Requests ─────────────────────────────────────
@mcp.tool()
def get_pull_requests(repository: str, state: str = "OPEN") -> str:
    """Get pull requests for a Bitbucket repository.
    Args:
        repository: Repository slug (e.g., 'sre-autopilot')
        state: PR state — OPEN, MERGED, or DECLINED
    """
    r = requests.get(
        f"{BBHOST}/repositories/{WORKSPACE}/{repository}/pullrequests",
        headers=auth_headers(),
        params={"state": state}, timeout=30)
    r.raise_for_status()
    prs = [{"id": x["id"], "title": x["title"],
            "author": x["author"]["display_name"],
            "source": x["source"]["branch"]["name"],
            "created_on": x.get("created_on", "")[:10]}
           for x in r.json().get("values", [])]
    return json.dumps(prs)

# ── Tool 3: Get Pipeline Status ───────────────────────────────────
@mcp.tool()
def get_pipeline_status(repository: str, limit: int = 5) -> str:
    """Get recent pipeline runs for a repository.
    Args:
        repository: Repository slug
        limit: Number of pipeline runs to return (default 5)
    """
    r = requests.get(
        f"{BBHOST}/repositories/{WORKSPACE}/{repository}/pipelines",
        headers=auth_headers(),
        params={"pagelen": limit}, timeout=30)
    r.raise_for_status()
    pipes = [{"build": x.get("build_number"),
              "state": x["state"]["name"],
              "result": x["state"].get("result", {}).get("name", ""),
              "branch": x.get("target", {}).get("branch", {}).get("name", "")}
             for x in r.json().get("values", [])]
    return json.dumps(pipes)

# ── Tool 4: Get Commit Diff ───────────────────────────────────────
@mcp.tool()
def get_commit_diff(repository: str, commit_hash: str) -> str:
    """Get changed files and line counts for a specific commit.
    Args:
        repository: Repository slug
        commit_hash: Full or short commit SHA
    """
    r = requests.get(
        f"{BBHOST}/repositories/{WORKSPACE}/{repository}/diffstat/{commit_hash}",
        headers=auth_headers(), timeout=30)
    r.raise_for_status()
    return json.dumps(r.json().get("values", []))

# ── Tool 5: Search Code ───────────────────────────────────────────
@mcp.tool()
def search_code(repository: str, query: str) -> str:
    """Search code in a Bitbucket repository.
    Args:
        repository: Repository slug
        query: Search query string (supports Bitbucket code search syntax)
    """
    r = requests.get(
        f"{BBHOST}/repositories/{WORKSPACE}/{repository}/search/code",
        headers=auth_headers(),
        params={"search_query": query}, timeout=30)
    r.raise_for_status()
    return json.dumps(r.json().get("values", []))

# ── Tool 6: Add PR Comment ────────────────────────────────────────
@mcp.tool()
def create_pr_comment(repository: str, pr_id: int, comment: str) -> str:
    """Add a review comment to a pull request.
    Args:
        repository: Repository slug
        pr_id: Pull request ID number
        comment: Comment text to post (supports markdown)
    """
    r = requests.post(
        f"{BBHOST}/repositories/{WORKSPACE}/{repository}/pullrequests/{pr_id}/comments",
        headers={**auth_headers(), "Content-Type": "application/json"},
        json={"content": {"raw": comment}}, timeout=30)
    r.raise_for_status()
    return json.dumps({"id": r.json().get("id"), "created": r.json().get("created_on")})
